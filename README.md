本项目是一个基于检索增强生成（Retrieval-Augmented Generation）架构的智能问答系统，旨在从文档中自动提取信息，构建知识库，并通过自然语言交互回答用户问题，系统集成了文档解析、数据清洗、向量检索、大语言模型生成等多个模块，形成一个完整的数据处理与问答工作流
用到的技术包括大模型DeepSeek通用模型，Chroma向量数据库与本地向量化模型，LangChain大模型应用开发框架
核心流程：原始文档→数据预处理→向量化存储→用户提问→检索相关文档→LLM生成答案→返回结果，更多技术细节可以参考CSDN系列文章“LangChain实战：新手手撸RAG全记录（一）”
项目本身功能比较基础，使用时可以将新增内容放入data文件中，大模型可以识别其中内容并作答，支持PDF、Word、Excel、TXT等格式的文件，也是我的第一个成熟的项目，后续的优化方向会放在构建完成的日志系统和更好的客户端上

====================================================================================================================

我用streamlit搭建web页面，并优化了输出逻辑，现在可以选择回答的简洁还是详细，详细的回答结果会包含信息来源、用时等信息，项目也变得更为完整了。做这个RAG文档问答项目，更像是在解决一连串小坑：每个环节单独看都不难，串起来却要反复试错，整体下来，最大的感受是RAG不是一个模型或一个库就能搞定的事，而是数据管道+检索策略+提示与工具的一整套设计。文档怎么切、怎么存、怎么限定了再查、怎么避免工具抢答，都会直接决定回答质量和用户感受。能跑通、能答对、来源还干净，那一刻还是挺有成就感的。
